## Project Notes

**OpenVine Cache Architecture**:
- VideoCacheManager: Uses SafeJsonCacheInfoRepository with key="openvine_video_cache"
- ImageCacheManager: Uses SafeJsonCacheInfoRepository with key="openvine_image_cache"
- Both extend flutter_cache_manager's CacheManager
- Both need to handle corrupted JSON gracefully

**Directory usage in OpenVine**:
1. getApplicationSupportDirectory() - Persistent data (Hive boxes, cache metadata JSON)
2. getTemporaryDirectory() - Temp files (video frames, recordings, bug reports)
3. getApplicationCacheDirectory() - Cache data

CacheRecoveryService clears all three directories when recovering from corruption.
