## Project Notes

**Fixed Tab Navigation: Notifications Tab Restored**

Rabble identified that during the router refactor, the notifications tab was lost and replaced with a hashtag tab. Hashtags should be a section within explore, not its own bottom nav tab.

**Changes Made:**

1. **route_utils.dart**: 
   - Added `RouteType.notifications` to the enum (positioned before profile)
   - Kept `RouteType.hashtag` for push routes from explore
   - Added parsing/building logic for `/notifications/:index` routes

2. **nav_extensions.dart**:
   - Added `goNotifications([int index])` extension method
   - Kept `goHashtag(String tag, [int index])` for hashtag push navigation

3. **app_router.dart**:
   - Replaced hashtag tab route with notifications tab route
   - Changed tab 2 from `/hashtag/:tag/:index` to `/notifications/:index`
   - Moved hashtag to non-tab push route (accessible from explore)
   - Updated `tabIndexFromLocation()` to map 'notifications' → index 2

4. **app_shell.dart**:
   - Updated `_titleFor()` to show "Notifications" for notifications tab
   - Updated `_routeTypeForTab()` to map index 2 → RouteType.notifications
   - Updated `_handleTabTap()` to call `context.goNotifications()` for tab 2
   - Updated bottom nav icon from `Icons.tag`/`Tags` to `Icons.notifications`/`Notifications`

5. **active_video_provider.dart**:
   - Added `RouteType.notifications` case to switch (returns null - no video feed)
   - Fixed exhaustive switch warning

**Final Tab Structure:**
- Tab 0: Home
- Tab 1: Explore
- Tab 2: **Notifications** (restored!)
- Tab 3: Profile

**Hashtag Navigation:**
- Still fully functional via `context.goHashtag(tag)` 
- Accessible as push route from explore screen
- Not a bottom nav tab anymore (correct architecture)

**Testing**: Flutter analyze passes (only expected deprecation warnings remain).
