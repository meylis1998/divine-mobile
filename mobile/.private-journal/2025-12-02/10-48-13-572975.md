## Project Notes

**Zendesk REST API Implementation (2025-12-02)**

Added Zendesk REST API support for platforms without native SDK (macOS, Windows, Web).

**Why needed:**
- Zendesk native SDK only works on iOS/Android via MethodChannel
- On macOS, `MissingPluginException` caused fallback to awful file share dialog
- User explicitly asked "why aren't we sending it to zendesk!@?!?!?"

**Solution:**
1. Added `ZENDESK_API_TOKEN` and `ZENDESK_API_EMAIL` to `zendesk_config.dart`
2. Added `createTicketViaApi()` and `createBugReportTicketViaApi()` to `ZendeskSupportService`
3. Updated `BugReportService.sendBugReport()` fallback chain:
   - Before: Worker API → File Share Dialog (blocking, awful UX)
   - After: Worker API → Zendesk REST API → Error message (no blocking dialog)

**Files Modified:**
- `lib/config/zendesk_config.dart` - Added apiToken, apiEmail, isRestApiConfigured
- `lib/services/zendesk_support_service.dart` - Added REST API methods
- `lib/services/bug_report_service.dart` - Updated fallback to use Zendesk API

**Key API Details:**
- Endpoint: `{zendeskUrl}/api/v2/requests.json`
- Auth: Basic auth with `{email}/token:{apiToken}`
- The Requests API allows ticket creation without end-user registration
